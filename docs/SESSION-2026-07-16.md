## 1. คลังยา sync สดข้ามเครื่อง (realtime)

**โจทย์:** พี่กันถามว่ามี Nortriptyline ใน Supabase ไหม → มี (id 350) แต่พิมพ์ "nor" ในแอปไม่เจอ
→ ไล่ดูโค้ด: ตัวกรอง `drugSearchText(d).includes(term)` ถูกต้อง · ต้นเหตุคือ **คลังยา cache ในเครื่องเก่า**
(แอปโหลดคลังยาครั้งเดียวตอน mount แล้ว cache `meddrp_drugs` · ยาที่เพิ่มทีหลังไม่เห็นจนกว่าจะรีเฟรช)
→ พี่กันรีเฟรชแล้วเจอ · สั่ง "ทำแบบไม่ต้องรีเฟรช"

**แก้:** ทำ realtime ตาราง `drugs` แบบเดียวกับ `incidents`
- `lib/data.ts` → `subscribeDrugs(cfg, onChange)` (1 channel `drugs-live` แยกจาก `incidents-live`)
- component → `refreshDrugs` (useCallback · ดึงคลังยาใหม่ทั้งชุด · อัปเดต state + cache เฉพาะตอนต่างจริง)
  + effect ใหม่ subscribe (debounce 500ms) + refetch ตอน `visibilitychange`/`online`
- migration `0008_enable_drugs_realtime.sql` (replica identity full + เพิ่ม `drugs` เข้า publication)
  → **applied ขึ้น Supabase แล้ว** (ตรวจ: `pg_publication_tables` เห็น drugs + incidents)

## 2. กราฟรายเดือน 6 → 12 เดือน + ปุ่มเลือกปี

- state `dashYear` (ค.ศ. · 0 = ปีปัจจุบัน) · กราฟโชว์ ม.ค.–ธ.ค. ของปีที่เลือกครบ 12 เดือน
- `yearOpts` = ปีที่มีข้อมูล (จาก `monthScopeRecs`) + ปีปัจจุบัน · เรียงใหม่→เก่า · ปุ่มปี พ.ศ. มุมขวาบนการ์ด
- หัวข้อ "จำนวนเคสรายเดือน · ปี 2569" · แท่งแคบลง (width 76%) gap ชิด · มือถือ font/gap เล็กลงให้พอดี 12 แท่ง

## 3. หน้ารายงานเพิ่มคอลัมน์ "รายละเอียด" (desktop + mobile)

- `recRows.detailText` = `detail` (ME) / `cause` (DRP) — ข้อความเหตุการณ์ที่พิมพ์
- เดสก์ท็อป: คอลัมน์ใหม่ก่อน "ผู้รายงาน" (max-width 260px · `-webkit-line-clamp:2` + `title` tooltip
  ตอน hover เห็นเต็ม) · ขยาย `min-width` ตาราง 860 → 1040
- มือถือ: บรรทัดล่างสุดของการ์ด (เส้นประคั่นด้านบน)

## 4. แก้ scroll หน้าผล "ส่งสำเร็จ"

- ปุ่ม "ส่งรายงานใหม่" กดแล้วค้างอยู่ล่างเพจ (ตำแหน่ง scroll เดิมก่อนส่ง)
- ต้นเหตุ: กดปุ่มแค่ set `result:null` · view คงเป็น `form` อยู่แล้ว → effect `scrollTo(0,0)` (key ที่ `view`)
  ไม่ทำงานเพราะ view ไม่เปลี่ยน
- แก้: onClick เรียก `window.scrollTo(0,0)` เองหลัง setState
