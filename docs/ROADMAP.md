## 🔴 1. ยุบฟอร์มแก้ไข ให้ใช้ฟอร์มเดียวกับหน้ากรอก ← งานถัดไป สำคัญที่สุด

**ปัญหา:** **หน้ากรอก** กับ **โหมดแก้ไข** เป็นโค้ดคนละชุดที่เขียนแยกกัน

| หัวข้อ | หน้ากรอก (`renderForm`) | โหมดแก้ไข (`renderEditMode`) |
|---|---|---|
| ประเภท Error / DRP | ปุ่มชิป + คำอธิบาย + ปุ่ม ⓘ ดูความหมาย | เมนู dropdown เปล่า ๆ |
| ชื่อยา | **ค้นหายาอัตโนมัติ** (คลังยา 417 ตัว) หลายแถว | ช่องพิมพ์ธรรมดา **ไม่มีตัวช่วยเลย** ← พี่กันเจอเอง 14 ก.ค. |
| บังคับกรอก | มี (ดาวแดง + ข้อความเตือน) | **ไม่มี** — ลบข้อมูลจำเป็นทิ้งแล้วบันทึกได้ |
| ระดับ NCC MERP | 4 โซนมีสี + ปุ่มดูความหมาย | dropdown |
| เงื่อนไขใหม่ (ผลตอบรับแพทย์ ฯลฯ) | ครบ | ต้องตามไปแก้ซ้ำทุกครั้ง = ตกขบวนเรื่อย ๆ |

**ผลเสีย:** ผิดกฎ "อะไรที่คล้ายกันต้องเป็นระบบเดียวกัน" · แก้ที่หนึ่งลืมอีกที่หนึ่งตลอด

**แผน:**
1. กด "✎ แก้ไขรายการนี้" → โหลดข้อมูลเคสเข้า `state.form` + ตั้ง `editingId`
2. ป๊อปอัปแสดง **`renderForm()` ตัวจริง** (ชิป · ค้นหายา · ปุ่มดูความหมาย · ติ๊กต่าง ๆ · เงื่อนไขครบ)
3. ปุ่มล่างเปลี่ยนเป็น "บันทึกการแก้ไข" / "ยกเลิก" (เรียก `saveEdit` เดิม · ยัง snapshot `history[]` เหมือนเดิม)
4. ลบ `renderEditMode` (~200 บรรทัด) ทิ้ง
5. ผลพลอยได้: บังคับกรอกมีผลตอนแก้ไขด้วย · หลังจากนี้ปรับหน้ากรอกที่เดียวพอ

## 🟠 2. ระบบแนบรูปผ่าน Cloudflare (ปุ่มแนบรูปตอนนี้ปิดอยู่)

- ตอนนี้ปุ่มแนบรูป = กล่องเทา + ป้าย "ยังใช้ไม่ได้" (`onAttachFile` ยังอยู่ในโค้ด รอเปิดใช้)
- เดิม: ย่อรูป → base64 → ฝังลงคอลัมน์ในตาราง = แฟ้มบวม โหลดหน้ารายงานช้าเมื่อเคสเยอะ
- **แผน (พี่กันสั่ง):** อัปโหลดขึ้น **Cloudflare** · แปลงเป็น **webp** · ลดความละเอียด + ลดขนาดไฟล์
- **ระบบนี้ทำครบแล้วใน TB Dashboard (รองรับทุกนามสกุล) → ยกโค้ดมาใช้ได้เลย**

## 🟡 3. จัดการรายชื่อผู้รายงาน (ค้างจาก session 10 ก.ค.)

- รายชื่อ 16 คนฝังตายใน `lib/constants.ts` (`REPORTERS`) — เพิ่ม/ลบคนต้องแก้โค้ด + deploy ใหม่
- **แผน:** ตาราง `reporters` ใน Supabase + หน้าจัดการในแท็บ "ตั้งค่า" (เพิ่ม / แก้ / ปิดการใช้งาน)
- ต้องเก็บใน Supabase เท่านั้น (localStorage จะไม่ sync ข้ามเครื่อง)

## 🟡 5. คลังยาให้อัปเดตข้ามเครื่องเอง

- แอปโหลดคลังยาครั้งเดียวแล้วเก็บในเครื่อง (`localStorage: meddrp_drugs`)
- เพิ่มยาใหม่ในฐานข้อมูล → **เครื่องที่เปิดค้างไว้ยังเห็นคลังเก่า** ต้อง Ctrl+Shift+R เอง (เจอตอนเพิ่ม SOF/VEL 14 ก.ค.)
- **แผน:** ทำ Realtime ให้ตาราง `drugs` แบบเดียวกับ `incidents` หรือเทียบเวลาอัปเดตล่าสุดตอนเปิดแอป
