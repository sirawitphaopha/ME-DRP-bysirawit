### 1. 🐛 บั๊กหลัก — "กรอกแล้วไม่เข้า Supabase" (เกิดทั้ง local + Cloudflare)
- **อาการ:** กดบันทึกแล้วข้อมูลไม่ขึ้นฐานข้อมูล
- **วินิจฉัย:** api log เจอ `POST /rest/v1/incidents → 400` · postgres log = `violates check constraint "incidents_outcome_check"`
- **ต้นเหตุ:** ฟอร์มส่ง `""` ให้ `severity`/`outcome` ที่มี CHECK constraint (Med ไม่มีช่อง outcome / DRP ไม่มีช่อง severity → ส่ง `""` ที่ไม่ผ่าน constraint)
- **แก้:** `lib/data.ts` → `toRow()` แปลง `severity`/`outcome` ที่เป็น `""` → `null` ก่อนส่ง

### 2. ระบบบังคับกรอกช่องจำเป็น (ดาวแดง `*` + เตือนสีแดง + บันทึกไม่ได้ถ้าไม่ครบ)
- **บังคับ:** HN · วันที่ · จุดที่พบ · ผู้รายงาน · Med(ประเภท Error · ลักษณะฯ · ความรุนแรง · รายละเอียดเหตุการณ์) · DRP(ประเภท DRP · สาเหตุ · Intervention · ผลลัพธ์ · รายละเอียด) · AN(เมื่อเลือก IPD)
- **ไม่บังคับ:** การแก้ไข/จัดการ · ชื่อยา · แนบรูป · ธงเตือนยา HAD/LASA

### 3. จุดที่พบ + IPD + AN
- `LOCATIONS` = ห้องยา OPD ทั่วไป / ห้องยา OPD NCD / **ห้องยา IPD** (เพิ่ม `IPD_LOCATION`)
- จุดที่พบมี**ทั้ง Med และ DRP** (เดิมมีแค่ Med) → `renderLocationField()` ใช้ร่วมกัน
- เลือก "ห้องยา IPD" → กรอก **AN (เลขที่ผู้ป่วยใน)** รูปแบบ `69-01234` (มีขีด · filter `[^0-9-]`)
- คอลัมน์ใหม่ `an` → `supabase/migrations/0002_add_an_column.sql` (รันที่ Supabase production แล้ว)

### 4. UI polish
- **ปุ่ม ⓘ ดูความหมาย** ที่ "ลักษณะความคลาดเคลื่อน" (state `showNatureLegend` · กางคำอธิบายทุกอัน เหมือน A–I)
