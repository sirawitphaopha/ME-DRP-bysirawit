### 1. ช่อง "ประเภท Error" เข้าหน้า DRP (งานหลัก)

พี่กันสั่ง copy ช่อง "ประเภท Error" (ขั้นตอนที่พลาด: Prescribing/Transcribing/
Pre-dispensing/Dispensing/Administration) จากฟอร์ม Med Error ไปหน้า DRP ให้ครบ
ทุกจุด เพราะ (ก) DRP มีในหน้ากดแก้ไขรายงานด้วย (ข) ต้องนับรวมใน Dashboard

**เคาะกับพี่กัน:** วางเป็น**ช่องแรกของ DRP** (ถัดจากหน่วยงาน/HN เหมือนหน้า ME ·
ไม่เอาบนสุดก่อนประเภท DRP) · **บังคับกรอกทั้ง Med และ DRP** · การ์ด Dashboard
นับรวมทุกอย่าง

แก้ครบ 6 จุด (`components/MedDrpApp.tsx`):
1. **ฟอร์มกรอก** — แยกบล็อกเป็นฟังก์ชันร่วม `renderErrorTypeField()` เรียกทั้ง
   `renderMedFields` (เดิม) และ `renderDrpFields` (ช่องแรก) — กันโค้ดซ้ำ
2. **โหมดแก้ไข** — สาขา DRP เพิ่มปุ่ม `efToggleArr("error_type")` (เดิมมีแค่ Med)
3. **หน้ารายละเอียด + ประวัติ** — โชว์แถว "ประเภท Error" ของ DRP (เดิม gate เฉพาะ Med)
4. **Dashboard** — `byErr` เอา `.filter(r=>r.type==="med")` ออก → นับ Med+DRP ·
   เปลี่ยนชื่อการ์ด "แยกตามประเภท Med Error" → "แยกตามประเภท Error"
5. **บังคับกรอก** — `validateIncident` ย้ายเงื่อนไข `error_type` ออกจากบล็อก
   `type==="med"` เป็นส่วนร่วม → บังคับทั้งคู่
6. **CSV + ค้นหา + DB** — ครอบ error_type ทุก type อยู่แล้ว · **ไม่มี migration**
   (คอลัมน์ `error_type` เป็น jsonb array ใช้ร่วม · `toRow` normalize เป็น array)

> หมายเหตุ: การ์ด Dashboard ยังเป็น**การ์ดสลับ** — กรอง "DRP อย่างเดียว" จะโชว์
> "แยกตามประเภท DRP" (10 หมวด) แทน · ดู "ทั้งหมด"/"Med" = โชว์ประเภท Error นับรวม ·
> ถ้าพี่กันอยากเห็น 2 การ์ดพร้อมกันตอนกรอง DRP ค่อยเพิ่มทีหลังได้

### 2. บั๊กเวรไม่ย้ายเช้า→บ่ายเอง (เปิดเว็บค้างทั้งวัน)

เดิมพอถึง 16:00 ระบบไม่ย้ายเวรจนกว่าจะกดรีเฟรช · เพิ่ม state `shiftAuto` (เริ่ม
`true`) · clock tick 1s เช็ค `shiftOf(เวลาปัจจุบัน)` ต่างจากที่ตั้ง → set
`occurred_time` ตามจริง (auto-follow) · กดเลือกเวรเอง = `shiftAuto:false` (หยุด
follow · เคารพที่เลือก) · ฟอร์มใหม่/สลับ ME↔DRP/บันทึกเสร็จ → reset `true`
