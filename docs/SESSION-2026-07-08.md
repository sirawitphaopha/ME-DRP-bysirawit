## การตัดสินใจสำคัญ

- **Next.js vs Vite** → เลือก Next.js เพราะใช้ในองค์กร + มีข้อมูลผู้ป่วย (เก็บ secret ฝั่ง server ได้, ต่อยอด auth ได้) และ host บน Cloudflare
- **Supabase** → ใช้โปรเจกต์เดิม tb-calculator (ตามที่ผู้ใช้สั่ง "เอาเข้า TB calc") ไม่สร้างใหม่
- **คงดีไซน์ pixel-perfect** → ทำ `css()` แปลง inline-style string ของดีไซน์ตรง ๆ + wrapper hover/focus แทนการรื้อเป็น CSS ใหม่
- **error_nature / drugs เป็น array (jsonb)** รองรับ multi-select + หลายยา (โค้ดยัง backward-compatible กับข้อมูล string เดิม)
- **loadRecords ไม่บล็อกด้วย network**: โชว์ข้อมูลในเครื่องทันที แล้ว refresh จาก Supabase เบื้องหลัง

## หมายเหตุ / ข้อจำกัดที่พบ

- **Network egress ของ sandbox บล็อก Supabase** (`Host not in allowlist`) — จึงทดสอบดึงข้อมูลจริงจากคอนเทนเนอร์ไม่ได้
  แต่ในเบราว์เซอร์ผู้ใช้/บน Cloudflare จะเข้าถึง Supabase ได้ตามปกติ (โค้ด fallback ทำงานถูกต้องเมื่อ network ล้ม)
- RLS ปัจจุบันเปิด anon อ่าน/เขียน (เดโม) — **ต้องรัดกุมก่อนใช้จริงกับข้อมูลผู้ป่วย** (ดู README ส่วนความปลอดภัย)
