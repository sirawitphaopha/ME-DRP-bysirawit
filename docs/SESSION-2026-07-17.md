## ที่มา
- พี่กันอยากบันทึกว่าเคสมี "ต้นตอ" จากวิชาชีพไหน (แพทย์/พยาบาล/เภสัช/แล็บ ฯลฯ)
- เดิมมีแต่ "ประเภท Error" ที่เป็น **ขั้นตอน** (Prescribing/Dispensing/...) ซึ่ง **ไม่ตรงกับวิชาชีพ 1:1**
  (พยาบาลบางทีคีย์คำสั่ง=prescribe · IPD เภสัชเป็นคน transcribe) → จึงแยกเป็นช่องใหม่
- พี่กันเคาะ: ชื่อ "หน่วยงานต้นเหตุ" · แยกช่อง · เลือกหลายอัน · บังคับกรอก · ทั้ง Med+DRP · เพิ่มกราฟ Dashboard

## ที่ทำ
- **`SOURCE_UNITS`** (`lib/constants.ts`) = 8 ตัวเลือก: แพทย์ · พยาบาล · เภสัชกร · ผู้ช่วยเภสัช · ห้องแล็บ ·
  เวชระเบียน · ผู้ป่วย / ญาติ · อื่น ๆ
- **data model:** `Incident`/`FormState` เพิ่ม `source_units` (jsonb array) + `source_unit_other` (text) ·
  `emptyForm` ตั้งค่าเริ่มต้น · `COLS`/`toRow` normalize array (เหมือน error_nature)
- **หน้ากรอก:** `renderSourceUnitsField()` (ชิปเลือกหลายอัน + "อื่น ๆ" พิมพ์ระบุ + error) วางถัดจาก "จุดที่พบ"
- **โหมดแก้ไข:** `renderEfSourceUnits()` (ใช้ `efArr`/`efToggleArr("source_units")`) วางหลัง AN ทั้ง Med/DRP
- **handler `toggleSourceUnit(k)`** mirror `toggleNature` (เลิกเลือก "อื่น ๆ" ล้างข้อความ · ลบ errors) ·
  `efToggleArr` ก็ล้าง `source_unit_other` เมื่อเลิกเลือก "อื่น ๆ"
- **บังคับกรอก:** `validateIncident` เพิ่มเงื่อนไข source_units อย่างน้อย 1 (ทั้ง Med/DRP) + "อื่น ๆ" ต้องระบุ
- **แสดงผล:** หน้ารายละเอียด (แถว "หน่วยงานต้นเหตุ" = `natureText(source_units, source_unit_other)` ทั้ง Med/DRP) ·
  `matchSearch` hay (ค้นด้วยชื่อหน่วยได้) · `exportCsv` (2 คอลัมน์ใหม่)
- **Dashboard:** `byUnit`/`unitBreak` นับแบบ array (1 เคสนับได้หลายหน่วย · Med+DRP · respect ตัวกรอง) +
  การ์ดกราฟใหม่ **"แยกตามหน่วยงานต้นเหตุ"** (gradient teal) ถัดจาก "แยกตามผู้รายงาน"
