## 1. Realtime — ข้อมูลสดข้ามเครื่อง (โจทย์หลักของ session)

**อาการ:** หลายเครื่องใช้พร้อมกัน เครื่อง A กรอกเคส เครื่อง B ไม่เห็น ต้องรีเฟรชเอง

**ต้นเหตุ (2 ชั้น):**
1. ตาราง `incidents` **ไม่ได้อยู่ใน publication `supabase_realtime` เลย** (คิวรีเช็คแล้ว ไม่มีตารางไหนอยู่เลยสักตาราง) → Supabase ไม่ส่ง event ออกมาตั้งแต่แรก
2. ฝั่งแอปไม่มีตัวรับสัญญาณ — โหลดข้อมูลแค่ตอน mount ครั้งเดียว

**แก้:**
- `supabase/migrations/0004_enable_realtime.sql` — `replica identity full` + `alter publication supabase_realtime add table incidents` (รันบน production แล้ว)
- `lib/data.ts` → `subscribeIncidents(cfg, onChange, onStatus)` — 1 channel ต่อ 1 ตาราง (`incidents-live`)
- `MedDrpApp.tsx` → `refreshRecords()` ดึงข้อมูลใหม่ทั้งชุดผ่าน API (ไม่ patch จาก payload) · เทียบก่อนว่าต่างจริงค่อย setState (เครื่องที่บันทึกเองไม่เด้ง toast ซ้ำ) · ถ้าเปิดหน้ารายละเอียดเคสที่ถูกแก้จากอีกเครื่องอยู่ → อัปเดตให้ด้วย
- debounce 400ms + refetch ตอน `visibilitychange` / `online` (กันสัญญาณหลุด)

**เทสจริง (ไม่แตะเบราว์เซอร์):** insert แถวเข้า DB → หน้าเว็บที่เปิดค้างไว้ขึ้นเอง 4 → 11 รายการ · ลบแถว → เหลือ 10 เอง ✓

## 3. DRP — ยกเครื่อง (พี่กันจับได้ว่าหมวดเดิมไม่ได้อิงตำรา)

**บทเรียน:** แคลร์เติมวงเล็บอังกฤษให้หมวดเดิม 7 อันโดยไม่เช็คว่าครบตามหลักวิชาการไหม → พี่กันทัก "เอามาตามตำราป่าวเนี่ย มันเยอะกว่านี้" → เสนอ Hepler & Strand vs PCNE ให้เลือก → พี่กันเลือกคลาสสิก

- `DRP_TYPES` 7 → **10 หมวด (Hepler & Strand)** — เพิ่ม Untreated indication, Drug use without indication, แยกขนาดยาเป็น Subtherapeutic/Overdosage, Adherence → Non-adherence
- **`KeyDesc.label`** = ป้ายที่โชว์ (ไทย + วงเล็บอังกฤษ) แยกจาก **`key`** = ค่าที่บันทึกจริง → เปลี่ยนข้อความบนจอได้โดยเคสเก่าไม่เพี้ยน · helper `drpLabel(key)` ใช้ทุกจุดแสดงผล (ตาราง/รายละเอียด/ประวัติ/Dashboard/ตัวกรอง/โหมดแก้ไข)
- ปุ่ม **ⓘ ดูความหมาย** (`showDrpLegend`) กางคำอธิบายครบ 10 หมวด เหมือนฝั่ง ME
- **ยุบช่องซ้ำซ้อน:** "สาเหตุของปัญหา" + "รายละเอียดเพิ่มเติม" → ช่องเดียว "รายละเอียดเหตุการณ์ / สาเหตุ" (ยังเก็บที่คอลัมน์ `cause` · เคสเก่าที่มี `detail` ยังโชว์ในหน้ารายละเอียด)
- **ติ๊ก "เภสัชกรแก้ไขเอง ไม่ผ่านแพทย์"** (`pharmacist_only` · `0006_add_pharmacist_only.sql`) → ซ่อนช่อง "ผลตอบรับจากแพทย์" + ไม่บังคับกรอก · หน้ารายละเอียดโชว์ป้าย "✓ เภสัชกรแก้ไขเอง"
- 🐛 **แก้บั๊กเงียบ:** `emptyForm` เดิม set `intervention = INTERVENTIONS[0]` ค้างไว้ → ถ้าลืมเลือกจะบันทึก "ปรึกษาแพทย์ผู้สั่งใช้" ไปเงียบ ๆ ทั้งที่ติดดาวแดงว่าบังคับ (validation ไม่มีผลจริง) → เปลี่ยนเป็นค่าว่าง + ตัวเลือก "— เลือกการแก้ไข —" + ข้อความเตือน
- ป้ายใหม่: "การ Intervention" → "การแก้ไข (Intervention)" · "ผลลัพธ์การ Intervention" → "ผลตอบรับจากแพทย์" · "ประเภทปัญหา DRP" → "ประเภทปัญหาจากการใช้ยา (DRP)"
